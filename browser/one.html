浏览器渲染流水线可视化

点击下方按钮，观察不同操作如何触发不同的渲染路径。

网络 & 浏览器进程

👨‍💻 CPU

1. 网络请求

用户输入URL，浏览器进程通过网络线程请求HTML文件。

渲染器进程 (主线程)

👨‍💻 CPU

2. 解析 (Parse)

HTML → DOM
CSS → CSSOM

👨‍💻 CPU

3. 样式 (Style)

合并DOM和CSSOM，生成渲染树 (Render Tree)。

JS 操作
el.style.width
👨‍💻 CPU

4. 布局 (Layout / Reflow)

计算每个节点的几何信息 (位置、大小)。
开销大

JS 操作
el.style.color
👨‍💻 CPU

5. 绘制 (Paint / Repaint)

生成绘制指令清单。
开销中等

6. 提交 (Commit)

将图层信息和绘制指令提交给合成器线程。

合成器线程 & GPU

JS 操作
el.style.transform
👨‍💻 CPU

7. 合成 (Composite)

合成器线程接收信息，准备进行光栅化和合成。
开销低

🚀 GPU

8. 光栅化 & 显示

GPU将图层转换为像素，并合成最终画面显示在屏幕上。
